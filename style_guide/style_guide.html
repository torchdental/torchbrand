Modals

Modals are used to force the user through a necessary workflow before continuing elsewhere. A good modal will have the following elements:

  • An option to back out, either with an X in the top right corner and/or a cancel button.
  • A title that briefly describes the main purpose of the modal.
  • A body the goes into more detail on what the ramifications of taking action are or explaining what already happened that caused this modal to appear.
  • Options, usually in the form of buttons, to continue or back out at the bottom of the modal.

Tables

Use tables to organize and display tabular data. There are several helper classes that can be used with tables:

  • .table-line-above: adds a line above tfoot
  • .table-line-below: adds a line below tr
  • .table-right-align-col: aligns data to the right in td, th
  • .table-shrink-width: reduces the width of td, th to the minimum to fit contents

Forms

A good form should have the following elements:

  • A header or title
  • Context on what the information is for
  • Large inputs
  • Validation messages for each input
  • Clear labels for each input
  • Optional more information for each input
  • A clear 'submit' button, that conveys what happens after pressing
  • An optional confirmation before submitting
  • A loading or processing display if there is async behavior
  • Feedback for after the form is submitted (success and error)

Buttons

Use buttons to indicate a non-navigational action that the use may take.

No Action Button conveys that there will be NO consequences from clicking this button, aside from display state. Best for 'close', 'cancel' or 'ok' buttons button
Secondary Action Button conveys that there will be consequences from clicking this button, though not the main expected action .btn-secondary
Primary Action Button conveys that there will be consequences from clicking this button, this is the main action .btn-primary
Small Button Variation shrinks the button for vertically tight places .btn-sm
Large Button Variation enlarges the button .btn-lg
Tight Button Variation shrinks the button to be only as wide as necessary .btn-tight


Disabled Button State indicates that the button is disabled and is not clickable :disabled
